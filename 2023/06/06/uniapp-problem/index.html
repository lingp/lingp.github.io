    使用Uniapp遇到的问题和坑，版本：vue3，HbuilderX-3.8.4.20230531

    1.字体文件引入不成功？

    参考链接：https://uniapp.dcloud.net.cn/tutorial/project.html
    static:存放应用引用的本地静态资源（如图片、视频、图标字体文件等）的目录，注意：静态资源只能存放于此

    2.使用nvue时，slot引入图标，在app不生效？

    在非App下slot正常使用，这种用法在app上不生效

    父：<text class="iconfont"><slot></slot></text>
子：<IconButton>&#xe848;</IconButton>

    在app上生效的方式，在子组件定义一个icon, 父组件传值进来’\ue848’，单引号不能缺失，注意如下

    父：<text class="iconfont">{{icon}}</text>
子：<IconButton :icon="'\ue848'"></IconButton>

    完整代码

    <!-- 父组件 -->
<view class="flex flex-row align-center justify-center mr-2">
    <IconButton :icon="'\ue848'">&#xe848;</IconButton>
    <IconButton :icon="'\ue84f'">&#xe84f;</IconButton>
</view>

    <!-- 子组件 -->
<template>
    <view class="flex align-center justify-center" hover-class="bg-hover-light" style="height: 80rpx; width: 80rpx;">
        <!-- #ifdef APP-PLUS-NVUE -->
            <text class="iconfont" style="font-size: 50rpx;color: #000000;">{{icon}}</text>
        <!-- #endif -->
        
        <!-- #ifndef APP-PLUS-NVUE -->
            <text class="iconfont" style="font-size: 50rpx;color: #000000;"><slot></slot></text>
        <!-- #endif -->
    </view>
</template>

<script setup>
const props = defineProps({
  icon: String
})

</script>

    2.使用nvue时，slot在微信小程序不渲染的问题？

    子组件定义

    <text v-if="isShowTitle"><slot name="title"></slot></text>

    有问题的代码

    <!-- 父组件 -->
<navBarVue :isShowTitle="true">
    <!-- 这样微信小程序不展示 -->
    <template v-slot:title>WeChat</template> 
</navBarVue>

    解决方式

    <!-- 父组件 -->
<navBarVue :isShowTitle="true">
    <!-- 这里多加一个<text></text>不然小程序不渲染，真是奇怪 -->
    <template v-slot:title><text>WeChat</text></template> 
</navBarVue>

    3.微信小程序端不支持计算属性绑定对象给 class、style

    这样是不行的，渲染后的元素节点显示 object。

    <template>
  <view class="dictionary-picker">
    <view :class="boxStyle"></view>
  </view>
</template>

<script>
export default {
  computed: {
    boxStyle() {
      return {
        active: this.activeTab ? true : false,
      };
    },
  },
};
</script>

    解决方法：不通过计算属性，直接在属性上绑定逻辑。

    <template>
  <view class="dictionary-picker">
    <view :class="[activeTab ? 'active' : '']"></view>
  </view>
</template>

    4. opacity: 0; 在uniapp中，点击事件不生效，但是background-color: rgba(0, 0, 0, 0)则可以。

    发现这个问题的起因：为了将蒙版透明，但是需要点击关闭的事件生效，发现opacity: 0;不可以，但是background-color: rgba(0, 0, 0, 0)可以

    <!-- 正常 opacity: 0; 在网页中是生效的，但是在uniapp不生效，如下 -->
<!DOCTYPE html>
<html lang="en">
<head>
    <meta charset="UTF-8">
    <meta name="viewport" content="width=device-width, initial-scale=1.0">
    <title>Document</title>
</head>
<body>
    <div id="test" style="height: 100vh; width: 100vw; background-color: red;
    opacity: 0;
    ">
        1
    </div>
</body>
<script>
    window.onload = function() {
       let obj = document.getElementById("test");
       obj.onclick = function() {
         alert("hello world;")
       }

    }
</script>
</html>

<!-- uniapp 生效代码 -->
<template>
    <!-- 弹出层 -->
    <view style="z-index: 100;overflow: hidden;" v-if="status">
        <!-- 蒙版 -->
        <view v-if="mask" class="mask-main"  :class="{transparent: isTransparent}"  @click="hide"></view>
        <!-- 弹出框内容 -->
        <view style="position: fixed; z-index: 9999;" :style="`top:${positionY}px; left:${positionX}px`">
            <slot></slot>
        </view>
    </view>
</template>

<script setup>
const props = defineProps({
    "mask": {
        type: Boolean,
        default: true
    },
    // 弹出层内容宽度
    "bodyWidth":{
        type:Number,
        default:0
    },
    // 弹出层内容高度
    "bodyHeight":{
        type:Number,
        default:0
    },
    // tabbar高度
    "tabbarHeight":{
        type:Number,
        default:0
    },
  // 蒙版是否透明
    "isTransparent": {
        type: Boolean,
        default: false
    }
})

const hide = () => {
    status.value = false
}

</script>

<style scoped>
    .mask-main {
        position: fixed;
        left: 0;
        bottom: 0;
        right: 0;
        top: 0;
        background-color: rgba(0, 0, 0, 0.5);
        z-index: 1;
    }
    .transparent {
        background-color: rgba(0, 0, 0, 0) !important;
    }
</style>
